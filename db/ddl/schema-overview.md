# Schema Overview

This folder documents the SQL schema generated by SQLAlchemy models in `py/storage`.

- Postgres stores normalized serving entities and metadata.
- Partitioned Parquet under `data/lake` stores raw envelopes and analytical datasets.
- Alembic owns schema evolution.
- Operational ingestion metadata now lives in Postgres:
  - `ingestion_job_definitions`, `ingestion_job_runs`
  - `source_cursor_state`
  - `bronze_object_manifest`, `schema_registry`
  - `data_quality_checks`, `data_quality_results`
- F1 extended coverage tables now include:
  - `f1_team_radio_metadata`
  - `f1_starting_grid`
  - `f1_telemetry_index` for heavy `car_data` / `location` object indices
- Polymarket execution/microstructure metadata now includes:
  - `polymarket_market_status_history`
  - `polymarket_open_interest_history`
  - `polymarket_orderbook_levels`
  - `polymarket_ws_message_manifest`
- Cross-source reconciliation now distinguishes:
  - candidate mappings in `mapping_candidates`
  - manual overrides in `manual_mapping_overrides`
  - finalized mappings in `entity_mapping_f1_to_polymarket`
